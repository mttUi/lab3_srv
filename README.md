# Распределение между ворами:
Потоки работают пока coins > 1
Каждый вор может взять монету только если у него ≤ монет чем у напарника
Используется мьютекс для синхронизации
Завершение распределения:
При нечетном числе: останавливаемся когда 1 монета, отдаем ее покойнику
При четном числе: распределяем все монеты между ворами

# Условие остановки: 
while (coins > 1) - гарантирует что последняя монета останется
Равное распределение: Алгоритм обеспечивает одинаковое количество монет у воров
Обработка покойника: Последняя монета при нечетном количестве отдается покойнику
Синхронизация: Мьютекс защищает все операции с общими переменными
